\subsection{Local Consistency}

Local consistency in constraint programming refers to a property of constraint networks where each individual constraint, or small group of constraints, is satisfied with respect to the current domains of the involved variables. Rather than seeking a globally valid solution for the entire problem, local consistency techniques focus on eliminating values from variable domains that cannot participate in any valid solution for the constraint(s) being considered. Common forms of local consistency include node consistency (unary constraints), arc consistency (binary constraints), and path consistency (for more complex interactions). These techniques are used to prune the search space early, improving the efficiency of constraint solvers by reducing the number of possible value combinations that need to be explored. Local consistency is a foundational concept for more advanced consistency levels and is often a precursor to achieving global consistency in constraint satisfaction problems.

\subsection{Generalized Arc Consistency and Bound Consistency}

Generalized Arc Consistency (GAC) is a fundamental local consistency property in Constraint Programming that extends the concept of arc consistency to constraints involving more than two variables. A constraint is GAC if, for every variable in the constraint, each value in its domain can participate in at least one valid assignment of the other variables that satisfies the constraint. GAC ensures that no value is left in a variable’s domain that cannot be part of any solution involving that constraint, thus significantly reducing the search space.
Bound Consistency (BC) is a specialized form of consistency used primarily for numerical constraints. It focuses on maintaining consistency at the bounds (minimum and maximum values) of variable domains, rather than checking every value within the domain. BC ensures that, for each constraint, the lower and upper bounds of the variables can potentially participate in a valid solution. This approach is particularly efficient for continuous or large discrete domains, as it avoids the computational overhead of checking every possible value.

    \caption{Illustration of Generalized Arc Consistency and Bound Consistency on $x<y\leq z$}
    \label{fig:GACandBC}
\end{figure}
In this figure we illustrate \textit{generalized arc consistency} (GAC) above and \textit{bound consistency} (BC) below, for the CSP: $x<y\leq z$.
To illustrate the advantage of bound consistency we extended the domain from [1..3] to [1..300], making it 100 times bigger while keeping the same encoding.
For generalized arc consistency, we can see the enumerated domains of $X$, $Y$ and $Z$.
Across the domains of the variables, we link possible valuations satisfying the whole problem: for instance if $X=1$, we are connected to two valid assignments for $Y$ and $Z$.
Bound consistency on the other hand, only requires an encoding of the upper and lower bounds of the domain, for which it maintains consistency in the same way as GAC.
