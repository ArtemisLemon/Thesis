The VIATRA Solver is a framework designed for the automatic generation of well-formed instance models for domain-specific languages. 
It natively supports the creation of instance models for EMF metamodels, with the capability to extend existing models as needed. 
The generated models are presented as standard instances of the target domain, ensuring compliance with the well-formedness constraints specified in the input language. 
Currently, these constraints are defined using the VQL graph query language from the VIATRA query and transformation framework, though the framework is architected to incorporate OCL constraints in future updates.
The VIATRA Solver operates by translating the language specification into a logic problem, which is then solved by an underlying first-order logic solver to produce valid graph models. 
The framework automatically leverages a range of background solvers, including Alloy (with SAT solvers such as Sat4j and Minisat), the Z3 SMT solver, and a newly developed graph solver, eliminating the need for language developers to possess specialized solver expertise. 
% Notably, the graph solver excels in generating large-scale models with over 5,000 objects, and empirical measurements indicate that models produced by this solver are both more diverse and realistic compared to those generated by alternative solvers like Alloy.
In addition to model generation, the framework provides tools for visualizing output graph models and collecting runtime statistics from the solvers. It also features a dedicated editor that allows users to select the target domain and configure the generation process, streamlining the workflow for language developers and modelers.
