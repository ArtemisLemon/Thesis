In this chapter we'll be modeling the link between the UML CSP modeling the instance, and the OCL CSP modeling the constraint expressions.
We'll be considering expressions such as \inlineocl{src.prop} and \inlineocl{src.var(prop)}, which means also connecting with the interpretation of the \textit{constant expressions} which can be done independently.
When building the model for navigation we can assume to have access to all the information accessible to a \inlineocl{PropExp}, such as the domains for the variables of the \inlineocl{src} expression and the \inlineocl{PropertyTable} identified by \inlineocl{prop}.

We will first show and test the main model for this thesis, which preserves order.
Then we'll present a secondary model which only preserves multiplicity.
