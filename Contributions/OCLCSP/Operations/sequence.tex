In this section we'll consider OCL operations which can be applied to ordered collections, namely: sequence and ordered set.

% \newpage
\subsection{src.prepend(e)}
Consider the expression \inlineocl{src.prepend(e)}, where \inlineocl{src} is an expression resulting in a collection of objects or integers and \inlineocl{e} is an expression of resulting in an object or int: the output of the operation is the sequence starting with \inlineocl{e} followed by \inlineocl{src}.
% Formal Definition:
% \begin{equation}\label{def:prepend}
%     prepend(X,Y,exp) \iff
% \end{equation}
Reformulation using Global Constraints:
\begin{equation}\label{csp:prepend}
    prepend(X,Y,z):
    \begin{cases}
        Y = z\;\mathbin{\|}\;X
    \end{cases}
\end{equation}
In this model we simply need to prepend the variable to our array of variables.

Reformulation using Global Constraints:

    append(X,Y,z):

\newpage\subsection{src.insertAt(i,e)}
Consider the expression \inlineocl{src.insertAt(i,e)}, where \inlineocl{src} is an expression resulting in a collection of objects or integers, \inlineocl{i} is an expression resulting in an integer, and \inlineocl{e} is an expression of resulting in an object or int: the output is the sequence composed of the i-1 first values of \inlineocl{src}, then \inlineocl{e}, followed by the rest of \inlineocl{src}.
% Formal Definition:
% \begin{equation}\label{def:insertAt}
%     insertAt(X,Y,exp) \iff
% \end{equation}
Reformulation using Global Constraints:
\begin{equation}\label{csp:insertAt}
    insertAt(X,Y,z,p):
    \begin{cases}
        \forall i \in [1..|X|]
        ~~p'_i = i + \llbracket i \geq p \rrbracket\\
        element(x_i,Y,p'_i)\\
        element(z,Y,p)
    \end{cases}
\end{equation}
This model finds the positions of values of $X$ in $Y$.
Their position in $Y$ is unchanged if they're before the included element: $p'_i = i$, and incremented after the included element: $p'_i = i + \llbracket i \geq p \rrbracket$.

\newpage\subsection{src.subSequence(s,f)}
Consider the expression \inlineocl{src.subSequence(s,f)}, where \inlineocl{src} is an expression resulting in a collection of objects or integers and \inlineocl{s,f} are expressions of resulting integers giving the start and finish of the sub-sequence in the sequence: the output is the sub-sequence of \inlineocl{src} starting at s, ending at f.
% Formal Definition:
% \begin{equation}\label{def:subSequence}
%     subSequence(X,Y,exp) \iff
% \end{equation}
Reformulation using Global Constraints:
\begin{equation}\label{csp:subSequence}
    subSequence(X,Y,s,f):
    \begin{cases}
        X' = d\;\mathbin{\|}\;X\\
        p_1 = s+1\\
        p_i = (s+i)*\llbracket s+i\leq j\rrbracket\\
        element(y_i,X',p_i)
    \end{cases}
\end{equation}
In this model we constrain the position of the \textit{output values} in the source variables: $element(y_i,X',p_i)$.
Because we generally need to pack the output with dummies, those dummies need a position in the source list.
To guarantee the presence of the dummy value in the source, we make $X'$, which a variable instantiated to the dummy value, followed by $X$.

\subsection{src.last()}
Consider the expression \inlineocl{src.last()}, where \inlineocl{src} is an expression resulting in a collection of objects or integers: the output is the last element of \inlineocl{src}.
% Formal Definition:
% \begin{equation}\label{def:last}
%     last(X,Y,exp) \iff
% \end{equation}

Reformulation using Global Constraints:
\begin{equation}\label{csp:last}
    last(X,y):
    \begin{cases}
        size(X,p)\\
        element(y,X,p)
    \end{cases}
\end{equation}
Because in out encoding, dummies are packed at the tail end of an array of variables, to get the last value we need to use the size operation to identify the position of the last element.

\newpage\subsection{src.reverse()}
Consider the expression \inlineocl{src.reverse()}, where \inlineocl{src} is an expression resulting in a collection of objects or integers: the output of the operations is the sequence in reversed order.
% Formal Definition:
% \begin{equation}\label{def:reverse}
%     reverse(X,Y,exp) \iff
% \end{equation}

Reformulation using Global Constraints:
\begin{equation}\label{csp:reverse}
    reverse(X,Y):
    \begin{cases}
        size(X,s)\\
        p_i = (s-i)*\llbracket i \leq s\rrbracket + i*\llbracket \neg(i \leq s)\rrbracket\\
        element(x_i,Y,p_i)
    \end{cases}
\end{equation}
Because our collections are packed with dummies at the tail end, and we wish to keep them there, we reverse the positions values but not dummies.
To get the new position of a value, we find its distance to then end, and make that its distance from the start in the new list.
If variable holds the dummy value, the value is copied to the same position in the output.
