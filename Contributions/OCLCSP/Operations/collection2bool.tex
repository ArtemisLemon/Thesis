In this section we'll consider expressions which result in a single boolean.
It is common to know the resulting boolean at compile time, such as at the top of an invariant expression.
In the cases where the resulting truth value is known, we often have a simpler model. 

\subsection{equals and different}
Consider an expression of the form \inlineocl{exp = exp2}, where both expressions resolve to a collection. If one array is smaller than the other, we append variables instantiated to $d$.
To model equals and different in OCL, we can simply use a conjunction or disjunction.  
\begin{equation}\label{csp:coleq}
    coleq(X,Y):
    \begin{cases}
        \forall i \in [1..z]\\
        ~~ x_i = y_i
    \end{cases}
\end{equation}
\begin{equation}\label{csp:coldiff}
    coldiff(X,Y):
    \begin{cases}
        \exists i \in [1..z]\\
        ~~ x_i \neq y_i
    \end{cases}
\end{equation}

In expressions such as \inlineocl{(exp = exp2) implies exp3}, the equality between collections is no longer enforced, but checked. 
In other words, we need a reified version of this constraint model.
\begin{equation}\label{csp:coleq_reif}
    coleq(X,Y,z):
    \begin{cases}
        \forall i \in I\\
        ~~b_i \iff (x_i = y_i)\\
        minimum(B,z)
    \end{cases}
\end{equation}
In this first model for equals, we reify the individual equalities.
The reified value for this constraint model, is the minimum value for the reified equalities.

\begin{equation}\label{csp:coldiff_reif}
    coldiff(X,Y,z):
    \begin{cases}
        \forall i \in I\\
        ~~b_i \iff (x_i \neq y_i)\\
        maximum(B,z)
        % z \iff (z' > 0)
    \end{cases}
\end{equation}
In this model for not equal, we reify the individual inequalities.
The reified value for this constraint model, is the maximum value for the reified equalities.

For example, given the collections $X=[3,1,2,d,d]$ and $Y=[3,0,2]$, we pad the second with $d$, to make it the same length as the longer array: $Y'=[3,0,2,d,d]$.
Because $x_1=1$ and $y'_1=0$. These two collections are different.
If $y_1=1$, they would be equal.

When reifying, for each couple $(x_i,y'_i)$ we reify the equality to $b_i$:
$B=[1,0,1,1,1]$. The minimum of this array $z$ gives us the truth value of this reified constraint.


\newpage\subsection{includes(e)}
Consider the expression \inlineocl{src.includes(e)}, where \inlineocl{src} is an expression resulting in a collection of objects or integers, and \inelineocl(e) is an (expression resulting in an) object or an integer.
% Formal Definition:
% \begin{equation}\label{def:includes}
%     includes(X,y) \iff y \in X
% \end{equation}

Reformulation using Global Constraints:
\begin{equation}\label{csp:includes}
    includes(X,y,z):
    \begin{cases}
        count(y,X,c)\\
        z = \llbracket c>0 \rrbracket
    \end{cases}
\end{equation}
For this model, we simply count the number of times the element is found in the list.
The reified value of the constraint $z$, is equal to weather or not the count is greater than zero.

For example given $X=[3,1,3,2,d]$ for \inlineocl{src}, and $y=3$ for \inlineocl{exp}, the count $c$ is equal to the count of $y$ in $X$, here $c=2$.
Finally the output $z$ constrained to wether or not, $c>2$.

\newpage\subsection{includesAll(e)}
Consider the expression \inlineocl{src.includesAll(e)}, where \inlineocl{src} is an expression resulting in a collection of objects or integers, and \inelineocl(e) is (an expression resulting in) a collection of objects or an integers.
% Formal Definition:
% \begin{equation}\label{def:includesAll}
%     includesAll(X,y) \iff \llbracket x_i \neq d \rrbracket
% \end{equation}

Reformulation using Global Constraints:
\begin{equation}\label{csp:includesAll}
    includesAll(X,Y,z):
    \begin{cases}
        \forall y_i \in Y,\\
        ~~includes(X,y_i,z_i')\\
        minimum(Z',z)
    \end{cases}
\end{equation}

For example, given $X=[3,1,3,2,d]$ and $Y=[3,2]$, we use the previous $includes$ model to return an array of booleans $B=[1,1]$, representing the inclusion of each element.
The minimum of $B$ is the truth value of the reified constraint. 

\subsection{excludesAll(e)}
Consider the expression \inlineocl{src.excludesAll(e)}, where \inlineocl{src} is an expression resulting in a collection of objects or integers, and \inelineocl(e) is (an expression resulting in) a collection of objects or an integers.
% Formal Definition:
% \begin{equation}\label{def:excludesAll}
%     excludesAll(X,y) \iff \llbracket x_i \neq d \rrbracket
% \end{equation}

Reformulation using Global Constraints:
\begin{equation}\label{csp:excludesAll}
    excludesAll(X,Y,z):
    \begin{cases}
        \forall y_i \in Y,\\
        ~~excludes(X,y_i,z_i')\\
        minimum(Z',z)
        % z = \llbracket z'' > 0 \rrbracket
    \end{cases}
\end{equation}

For example, given $X=[3,1,3,2,d]$ and $Y=[4,2]$, we use the previous $includes$ model to return an array of booleans $B=[1,0]$, representing the exclusion of each element.
The minimum of $B$ is the truth value of the reified constraint.
Here because 2 isn't excluded from $X$, the minimum found in $B$ is 0, or false.

\newpage\subsection{forall(exp)}
Consider the expression \inlineocl{src.forall(exp)}, where the source \inlineocl{src} is an expression resulting in a collection of objects or integers and \inlineocl{exp} is an sub-expression of type boolean which applies to a sub-collection of the collection from the source \inlineocl{src}.
The expression as a whole resolves to true if all the sub-expressions resolves to true, and false otherwise.
% Formal Definition:
% \begin{equation}\label{def:forall}
%     forall(X,e,y) \iff
% \end{equation}

Reformulation using Global Constraints:
\begin{equation}\label{csp:forall}
    forall(X,y,exp):
    \begin{cases}
        itpred(X,B,exp)\\
        minimum(B,y)
        % y = \Pi B
    \end{cases}
\end{equation}
Our model for \inlineocl{forall} gathers a collection of booleans, and simply textit{ands} across them.

\subsubsection{Iterating with predicates}
The \inlineocl{forall} expression, along with \inlineocl{exists} and \inlineocl{one}, take an iterable expression which can take several iterators, such as \inlineocl{a,b | pred(a,b)} where \inlineocl{pred} is a predicate applied to a and b.
% In the OCL standard, \inlineocl{forall} takes up to two iterators,  

\begin{equation}\label{csp:itpred}
    itpred(X,B,exp):
    \begin{cases}
        % \forall b_{ijk...} \in B, \forall x_i,x_j,x_k,... \in X\\
        \forall i,j,k,... \in [1..|X|]\\
        ~~b_{ijk...} \iff (((x_i \neq d) \wedge (x_j \neq d) \wedge ...) \implies exp(x_i,x_j,x_k,...))
    \end{cases}
\end{equation}
In this first equation, we focus on the core of $itpred$ associating the input $X$ and the output $B$.
The number of variables in $B$ is equal to $(|X|)^z$ where $z$ is the number of iterators.
To index $B$ we'll use $z$ indices, ranging from 1 to $|X|$.
Each $b_{ijk...}$ , where i,j,k,.. are in [1..|X|], is associated to the combination variables $x_i,x_j,x_k,...,x_z$.
The $itpred$ constraint can be understood as a dispatcher, applying the expression $exp$ of every combination of variables from $X$.
Each combination of variables is associated to a boolean, which is false only if variables have valid values and the expression upon them is false.
% The objective here, is to only apply the expression if no variable the expression is applied to is equal to the dummy value of the domain.
To complete the model, lets consider the expression \inlineocl{src.forall(a,b,c| (a<b and b<c) implies (a+b=c))}, \inlineocl{exp} in the above would be replaced with:
$$(x_i<x_j \wedge x_j < x_k) \implies (x_i+x_j=x_k+x_k)$$

\subsection{one(exp)}
This operation can have at most one iterator variable.

Consider the expression \inlineocl{src.one(exp)}, where the source \inlineocl{src} is an expression resulting in a collection of objects or integers and \inlineocl{exp} is an sub-expression of type boolean which applies to the elements of the collection from the source \inlineocl{src}.
The expression as a whole resolves to true if one of the sub-expressions resolves to true, and false otherwise.
% Formal Definition:
% \begin{equation}\label{def:one}
%     one(X,exp,Y) \iff
% \end{equation}

Reformulation using Global Constraints:
\begin{equation}\label{csp:one}
    one(X,y,exp):
    \begin{cases}
        itpred(X,B,exp)\\
        count(\top,B,y')\\
        y = \llbracket y' = 1 \rrbracket
    \end{cases}
\end{equation}
